HTTP

HTTP is an application layer communication protocol that is used to access resources on the World Wide Web

HTTP was jointly developed by the W3C and the IETF

Version History

HTTP Fundamentals

The HTTP structure is based on the client-server architecture. A client establishes a connection with a server and sends a request to the server in the form of a request method, URI, and protocol version, followed by a MIME-like message containing request modifiers, client information, and possible body content. The server responds with a status line, including the message's protocol version and a success or error code, followed by a MIME-like message containing server information, entity metainformation, and possible body content.

  • Servers: also known as origin servers or where the the resources reside.
  • Clients: also known as user agents

HTTP uses a request-response protocol

HTTP is a stateless communications protocol

HTTP Resource Addressing

HTTP resources are identified using URLs:

URLs

Unique Identifier indicating the location of a specific resource.

PARTS OF A URL

Scheme

The scheme pertains to the protocol used by a website (i.e. HTTP, FTP, HTTPS).

Path

The path refers to resource path which is either static or dynamic

Query

Done using Key = Value pairs

Fragment Identifier

Identifies which fragment you are in the website. Starts with the pound (#) symbol.

HTTP Message

HTTP Request Message

  1. Request Line
    • Method
    • Request Target
    • Protocol Version
  2. Message Headers
    • Requires at least the "Host" header.
  3. Empty Line
    • An empty white line that is required.
  4. Message Body

HTTP Response Message

  1. Status Line
    • Protocol Version
    • Status Code
    • Reason Phrase
  2. Message Headers
    • Requires at least the "Host" header.
  3. Empty Line
    • An empty white line that is required.
  4. Message Body

Types of Message Headers

  1. General Purpose
    • Gives information that is general about the message.
  2. Request/Response
    • Specific to a request or response message.
  3. Entity
    • Pertains to the content.

HTTP Methods

GET

  • Transfers resource to a client.

HEAD

  • Identical to the GET method except it retrieves metadata about the target resource.

POST

  • Requests that the target resource perform resource-specific processing.

PUT

  • Creates or replaces target resource.

OPTIONS

  • Requests information about he communication options available for the target resource.

DELETE

  • Removal of resource.

TRACE

  • This method is used for testing or diagnostics. It also replicates the request in the response message body.

CONNECT

  • Establishes a tunnel between the server and client.

Categories of HTTP methods

SAFE METHODS

  • Read-only methods
  • GET, HEAD, OPTIONS, TRACE

IDEMPOTENT METHODS

  • Has the same effect when requested once
  • GET, HEAD, OPTIONS, TRACE, PUT, DELETE

CACHEABLE METHODS

  • The response to a method is allowed to be stored for future reuse
  • GET, HEAD, POST

Content Negotiation

Content negotiation occur when a resource may have multiple formats.

An HTTP client can negotiate which of the available resource formats they prefer. This can be seen in the Accept and Accept-Language header. On the server side, it is indicated by the Vary header.

Range Request

Range request occur when a client request for only a part of the resource (i.e. downloading) specified by a range. Range request information can be seen in the Accept-Ranges header.

Authorization/Authentication

Resources on the web can ask for user authentication before a client can access its contents or resources. Information regarding authentication or authorization can be seen in the WWW-Authenticate header.
